## 1. Purpose

Build a complete, graduate-level course support system covering all 14 teaching weeks + UTS + UAS of MST304. Outputs must support deep learning, presentation prep, and exam readiness, with explicit Indonesian corporate-context anchoring using verifiable primary sources. Final deliverables are HTML files designed for both digital study and clean Print-to-PDF conversion.

## 3. Design decisions (approved)

### 3.1 Gap-handling for missing chapters & articles
**Decision (user: A).** Weeks 12–14 (where TPGS Ch.11–12 do not exist in 7th ed.) are driven by the week's articles + crosswalk into Anthony Henry's *Understanding Strategic Management* (4e). For Art.5–29 where PDFs are not provided, produce structured deep-dive analysis from the public abstract + canonical secondary literature (e.g., Barney 1991 RBV, Kaplan & Norton 1992/2006 BSC, Porter frameworks) with a visible disclosure banner.

### 3.2 Indonesian company example depth
**Decision (user: C).** Each week gets **1 deep flagship (~1–1.5 pages)** anchored in verifiable publicly-disclosed data + **1–2 sidebar comparatives (~1–2 paragraphs each)** showing the same concept manifesting differently in another Indonesian firm. Sourcing is strict (see §5.3).

### 3.3 Visual style
**Decision (user: A) — Clean Editorial.**
- **Typography**: Sans-serif for headings (Inter or Geist); serif for body (Merriweather or Charter). Fallback stack for offline-safety.
- **Color palette**: One restrained accent — proposed deep navy `#1e3a8a` with a secondary muted teal `#0f766e` for callouts. Warm black `#111827` for ink; `#f8fafc` for subtle panel backgrounds.
- **Layout**: Fixed 220px left sidebar with week navigator + within-week TOC. Main content ~680–760px max-width for readability.
- **Elements**: Pull-quote callouts, concept boxes with 1px hairline rules, "Pemeriksa Konsep" check-boxes, SVG framework diagrams.
- **Print**: Sidebar auto-hides via `@media print`; main becomes full-width; content prints like a textbook page.

## 4. Architecture

### 4.1 Pipeline shape
**Approach (2) — Markdown-fragment + Rust shell builder.** Content is authored as Markdown fragments (one per section per week, with YAML front-matter). Rust ingests these, assembles them with minijinja HTML templates, emits final HTML to `course-materials/outputs/`. The Rust pipeline is purely a "shell builder": all prose lives in human-readable, diff-friendly Markdown.

### 4.2 Rust crate dependencies
Minimal, actively-maintained crates only:
- `pulldown-cmark` — Markdown → HTML
- `minijinja` — HTML templating (Jinja-compatible)
- `serde` / `serde_yaml` / `toml` — front-matter + config parsing
- `walkdir` — content-tree traversal
- `anyhow` + `thiserror` — error handling
- `clap` — CLI subcommands (`build`, `build-week <N>`, `clean`, `watch`)

### 4.3 Data model (Rust structs)
```rust
struct Week { num: u8, topic_id: String, topic_title_id: String, topic_title_en: String,
              tpgs_chapters: Vec<u8>, henry_chapters: Vec<u8>, articles: Vec<ArticleRef>,
              indo_flagship: Company, indo_sidebars: Vec<Company>, is_exam_prep: bool }
struct ArticleRef { number: u8, citation: Citation, pdf_available: bool, week_role: String }
struct Company { slug: String, name: String, idx_ticker: Option<String>,
                 concept_anchor: String, sources: Vec<SourceCitation> }
struct SourceCitation { doc_type: DocType, year: u16, section: Option<String>, relevance: String }
enum DocType { AnnualReport, IDXDisclosure, OJKFiling, InvestorRelations, SustainabilityReport, OfficialPublication }
```

## 5. Output specification

### 5.1 Weekly output anatomy (4 HTML files per week)

**① Study Guide / Aid** — `course-materials/outputs/Study Guide - Aid/Week NN/index.html`
- Reading order (what first, second, third)
- Central concepts map (3–5 load-bearing ideas, hierarchy diagram)
- Concept connections (prior-weeks ← this week → later weeks)
- Focus for discussion/presentation + likely UTS/UAS angles
- Pre-reading checklist + self-check questions

**② Main Summary / Ebook** — `course-materials/outputs/Main Summary - Ebook/Week NN/index.html`
- Chapter-by-chapter synthesis of TPGS (+Henry for W12–14, +KK where behaviorally relevant)
- Frameworks as SVG diagrams (PESTEL, Five Forces, Value Chain, VRIN, Diamond, Generic Strategies matrix, BSC, BCG, Ansoff, etc.)
- Bold key terms with English original in brackets: "keunggulan bersaing berkelanjutan *(sustained competitive advantage)*"
- "Pemeriksa Konsep" callouts
- Per-section citation blocks

**③ Article Analysis** — `course-materials/outputs/Articles/Week NN/index.html`
- Weeks 2–3: full deep analysis of Art.1–4 (research question, method, findings, theoretical contribution, critique, concept anchor into TPGS)
- Other weeks: structured analysis from public abstracts + canonical literature with disclosure banner

**④ Indonesian Company Examples** — `course-materials/outputs/Indonesian Company Examples/Week NN/index.html`
- Flagship (1–1.5 pages): narrative + 2–3 hard data points with citation + concept-anchored interpretation
- 1–2 sidebars (1–2 paragraphs)
- Sources table: Document type · Year · Section · Relevance

### 5.2 Additional outputs

- **Visual Companion** (`course-materials/outputs/Visual Companion/index.html`) — master mapping view: syllabus week × topic × TPGS ch × Henry ch × articles × Indonesian flagship; includes "revision-review" panels where sections need further attention; digital-first, not paginated for print.
- **Master Print Bundle** (`course-materials/outputs/Master Print Bundle/course-companion.html`) — single-file all-weeks PDF-ready: cover, TOC with dot leaders, per-week page-start, back-matter (combined glossary + combined sources index + complete Indonesian company index).

### 5.3 Indonesian sourcing protocol (strict)

1. **Company allowlist** — only firms widely documented in publicly-referenced ARs, IDX filings, OJK disclosures, and IR materials. Proposed allowlist: BCA, BRI, Mandiri, BNI, BSI, Astra International, Astra Otoparts, Telkom, Telkomsel, Unilever Indonesia, Indofood/ICBP, Mayora, Kalbe, Sido Muncul, GoTo Gojek Tokopedia, Bukalapak, Adaro Energy, Indika Energy, PTBA, Pertamina, PLN, Garuda Indonesia, Djarum Group, Gudang Garam, HM Sampoerna, ACE Hardware, Mitra Adi Perkasa, Ruangguru, Kopi Kenangan.
2. **Citation format** — every data point links to document type: `Astra International AR 2023, Segment Reporting (p. 142)`.
3. **Anti-fabrication rule** — if exact figure is not defensible, use descriptive language ("among top-3 by revenue") or ranges rather than invent numbers. No made-up percentages, dates, or direct quotes.
4. **Disclosure footer** on every Indonesian page: *"Data disarikan dari materi publik yang diungkapkan perusahaan Indonesia (AR, IDX, OJK, IR). Mahasiswa dianjurkan memverifikasi angka terkini dari pengajuan aktual."*

## 7. Print strategy

- Shared `print.css` across all outputs; A4 portrait; 2.5 cm margins.
- Sidebar `display: none` on print; main element becomes full-width.
- `@page` rule: running header "MST304 · Week NN · [Topic]"; page number bottom-right.
- `page-break-before: always` on H2 (major sections).
- `page-break-inside: avoid` on tables, framework boxes, citation blocks, case study boxes.
- Widow/orphan control: `orphans: 3; widows: 3`.
- SVG diagrams `max-width: 100%` with inlined styles (no external CSS dependency).
- Master Print Bundle: explicit cover page + TOC with dot leaders + per-week page-start + back-matter (glossary, sources).
- Verified by Chromium headless Print-to-PDF during QC phase.

## 9. Language & register

- **Student-facing primary prose**: Indonesian (Bahasa Indonesia akademik).
- **Strategic-management terms**: bold Indonesian term with English original in brackets on first occurrence per page.
- **Direct quotes from articles/textbooks**: preserved in original English with Indonesian gloss underneath.
- **Citations, references, tables of sources**: English (per academic convention).
- **Internal spec/architecture documents (this file)**: English.

## 10. Quality control criteria

- **Conceptual accuracy**: all TPGS/Henry content traceable to the source chapter; no conflation of frameworks.
- **Indonesian example verifiability**: every data point annotated with document type + year; anti-fabrication rule enforced.
- **Print fidelity**: Chromium Print-to-PDF produces output that matches on-screen layout with no awkward breaks in critical blocks.
- **Cross-referencing**: prior-week concepts linked; forward references noted.
- **Disclosure banners present** on all article-analysis pages where PDF was not provided.
- **Zero broken links** in any output HTML.

## 11. Out-of-scope (explicit)

- Live IDX/OJK data fetching (no real-time API calls during generation).
- PDF file output directly from Rust (user uses browser Print-to-PDF — this is deliberate for flexibility).
- Interactive JS in outputs beyond minimal nav/accordion (print-stability is the priority).
- Slide decks / presentation files (student's own task based on the summaries).
- Simulation-based content (TPGS's BSG/GLO-BUS).
